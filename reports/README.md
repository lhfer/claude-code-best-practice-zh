# Reports

这个目录负责解释“概念之间的边界”和“为什么上游仓会这样设计”。

如果说：

- `best-practice/` 是“这是什么”
- `implementation/` 是“它在仓库里怎么落地”

那 `reports/` 更像：

> “为什么要这么分、哪些概念容易混、哪些说法已经在漂移。”
