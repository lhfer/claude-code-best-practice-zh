# LOCALIZATION_POLICY

本文件定义这份中文 fork 的翻译边界、重编原则和同步规则。

## 目标

这不是逐句翻译工程，而是中文本土化重编工程。

我们的目标是：

- 让中国开发者能在更低认知负担下理解 Claude Code 的核心概念。
- 保持与 upstream 的事实一致性。
- 不破坏任何会影响运行行为的配置和示例。

## 一级原则

### 1. 执行层保持英文原样

以下内容默认不翻译、不改写语义：

- 代码块
- 命令名
- frontmatter key
- 环境变量
- 工具名
- 模型名
- 文件路径
- `.claude/`
- `.codex/`
- `.mcp.json`

理由很简单：这些东西一旦中文化，仓库就会从“可运行参考仓”变成“看起来很懂、实际上不好用的样板仓”。

### 2. 说明层允许彻底重编

以下内容可以按中文阅读习惯重写：

- `README.md`
- `best-practice/`
- `reports/`
- `implementation/`
- `orchestration-workflow/`
- 面向读者的 tips 摘要

这里的“重编”不是偷懒，而是刻意避免：

- 英文句式直译
- 海外语境照搬
- 术语只翻不解释
- 原仓结构适合作者、不适合中文读者

### 3. 人物观点保留出处，但不用直译腔

涉及 Boris / Thariq / Cat / Noah 等引用时：

- 保留人物名和原始来源链接
- 中文里写成“观点摘要”
- 必要时补一句译注解释语境

不做：

- 逐句金句翻译
- 硬保留英文修辞
- 把海外技术社区语气生搬进中文

## 中文写作规则

- 先讲人话，再上术语。
- 首次出现的关键术语给出中文解释和英文原名。
- 后续使用统一译法，遵循 [GLOSSARY.md](GLOSSARY.md)。
- 能一句讲清的，不写成三句。
- 不追求“翻得全”，追求“看得懂、记得住、用得上”。

## 同步规则

每次同步 upstream 时，按下面顺序处理：

1. 更新 [SYNC_STATUS.md](SYNC_STATUS.md)
2. 识别受影响模块
3. 区分两类变化：
   - `事实同步`：版本、链接、路径、能力变更
   - `中文重编`：面向读者的表达、结构、示例解释
4. 优先修正事实漂移
5. 最后再润色中文表达

## 提交规则

建议 commit message 显式标明类型：

- `sync:` 跟上游同步
- `docs:` 中文重编
- `glossary:` 术语统一
- `community:` issue / PR / contributing 相关改进

## 不做的事

- 不把 `.claude/` 里的命令正文翻成中文
- 不把 YAML key 改成中文
- 不为了“本土化”去替换运行示例的核心逻辑
- 不制造一个看似中文友好、实际上无法运行的假仓库
